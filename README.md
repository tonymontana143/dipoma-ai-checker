# 🤖 Детекция токсичных комментариев для социальных медиа

**Тема дипломной работы**: Development of toxic comment detection methods for social media platforms

## 📌 Описание проекта

Этот проект реализует BERT-based модель для классификации токсичных комментариев в русскоязычных социальных сетях. Модель обучается на датасете русских комментариев и использует **RuBERT** - специально обученную версию BERT для русского языка.

### 🎯 Основные возможности

- 🇷🇺 Полная поддержка русского языка
- 📊 Бинарная классификация (токсичный/безопасный)
- ⚡ Быстрое обучение и предсказание
- 📈 Отслеживание метрик обучения
- 🔧 Простая интеграция в приложения

## 📊 Датасет

**Файл**: `labeled.csv`

**Структура**:
- `comment` - текст комментария на русском языке
- `toxic` - метка (1.0 = токсичный, 0.0 = безопасный)

**Размер**: ~28,000 комментариев

**Пример**:
```csv
comment,toxic
"Спасибо за статью!",0.0
"Ты идиот!",1.0
"Как дела?",0.0
"Я тебя ненавижу!",1.0
```

## 🤖 Модель

**Название**: DeepPavlov/rubert-base-cased

**Характеристики**:
- Специально обучена на русском языке
- 109 миллионов параметров
- Case-sensitive (учитывает прописные/строчные буквы)
- Проверенная эффективность на задачах классификации текста

## 📈 Метрики качества

### Ожидаемые результаты после полного обучения

| Метрика | Значение | Описание |
|---------|----------|---------|
| **Accuracy** | > 0.92 | Доля правильных предсказаний |
| **F1 Score** | > 0.85 | Гармоническое среднее Precision и Recall |
| **ROC-AUC** | > 0.95 | Area Under ROC Curve |

## ⚙️ Параметры обучения

```python
# Конфигурация модели
CONFIG = {
    'model_name': 'DeepPavlov/rubert-base-cased',  # Русский BERT
    'max_length': 128,                              # Макс. длина токенов
    'batch_size': 8,                                # Размер батча
    'epochs': 3,                                    # Количество эпох
    'learning_rate': 2e-5,                          # Скорость обучения
    'num_labels': 1,                                # Бинарная классификация
}
```

### Оптимизатор

- **Алгоритм**: AdamW
- **Learning Rate Schedule**: Linear
- **Warmup Steps**: 500
- **Weight Decay**: 0.01

## 🆘 Решение проблем

### ❌ Out of Memory (OOM)

**Проблема**: 
```
RuntimeError: CUDA out of memory
```

**Решение**: Уменьшить batch_size в `train.py`:
```python
CONFIG['batch_size'] = 4  # вместо 8
```

### ❌ Модель не загружается

**Проблема**: 
```
FileNotFoundError: Model not found
```

**Решение**: Первый раз модель автоматически скачается с Hugging Face Hub. Убедитесь в интернет-соединении.

### ❌ Низкие метрики

**Решение**:
1. Увеличить количество эпох до 5
2. Использовать больший датасет
3. Увеличить max_length до 256
