# 📦 Установка ToxicShield Universal Extension

Пошаговая инструкция по установке расширения для Google Chrome, Microsoft Edge и других Chromium-браузеров.

---

## 📋 Предварительные требования

### 1. Backend API должен быть запущен

**Вариант А: Локальный Docker запуск (рекомендуется)**
```bash
cd toxic-api
docker-compose up -d
```

Проверьте, что API работает:
```bash
curl http://localhost:8000/health
# Должен вернуть: {"status":"healthy","model":"openai","model_loaded":true}
```

**Вариант Б: Удаленный сервер**
- Если вы развернули API на удаленном сервере (DigitalOcean, AWS, etc.)
- Запомните URL вашего API (например, `https://your-domain.com/api/check`)
- Вы укажете этот URL в настройках расширения после установки

### 2. Файлы расширения

Убедитесь, что у вас есть папка `toxic-shield-extension` с файлами:
```
toxic-shield-extension/
├── manifest.json
├── content.js
├── background.js
├── popup.html
├── popup.js
├── styles.css
├── icons/
│   └── (иконки будут созданы на шаге 2)
├── README.md
└── INSTALLATION.md
```

## 🎨 Шаг 1: Создание иконок (опционально)

Расширению нужны иконки в 3 размерах: 16x16, 48x48, 128x128 пикселей.

### Вариант A: Создать иконки автоматически

**С помощью ImageMagick:**
```bash
cd toxic-shield-extension/icons

# Создаем простую иконку со щитом (emoji)
convert -size 128x128 xc:white \
  -font Arial -pointsize 80 -fill '#667eea' \
  -gravity center -annotate +0+0 "🛡" \
  icon128.png

# Создаем меньшие размеры
convert icon128.png -resize 48x48 icon48.png
convert icon128.png -resize 16x16 icon16.png
```

**С помощью Python + Pillow:**
```bash
cd toxic-shield-extension

python3 << 'EOF'
from PIL import Image, ImageDraw, ImageFont

def create_icon(size):
    # Создаем градиентный фон
    img = Image.new('RGB', (size, size), color='white')
    draw = ImageDraw.Draw(img)
    
    # Рисуем круг с градиентом
    for i in range(size):
        color = (102 + i * 30 // size, 126 + i * 60 // size, 234)
        draw.ellipse([i//4, i//4, size-i//4, size-i//4], fill=color)
    
    # Сохраняем
    img.save(f'icons/icon{size}.png')
    print(f'Created icon{size}.png')

create_icon(128)
create_icon(48)
create_icon(16)
EOF
```

### Вариант B: Использовать готовые иконки

1. Создайте иконки в любом графическом редакторе (Figma, Photoshop, GIMP)
2. Используйте данный дизайн:
   - Фон: градиент от `#667eea` до `#764ba2`
   - Символ: белый щит 🛡 или буква "T"
3. Экспортируйте в размерах 16x16, 48x48, 128x128 px
4. Сохраните в папку `icons/` с именами `icon16.png`, `icon48.png`, `icon128.png`

### Вариант C: Временные placeholder иконки

Если нужно быстро протестировать, создайте простые цветные квадраты:

```bash
cd toxic-shield-extension/icons

# Создаем простые цветные квадраты
convert -size 128x128 xc:'#667eea' icon128.png
convert -size 48x48 xc:'#667eea' icon48.png
convert -size 16x16 xc:'#667eea' icon16.png
```

**После создания проверьте:**
```bash
ls -lh icons/
# Должно быть: icon16.png, icon48.png, icon128.png
```

## 🔧 Шаг 2: Установка в Chrome

### 1. Откройте страницу расширений

**Способ 1 (через адресную строку):**
- Введите в адресную строку: `chrome://extensions/`
- Нажмите Enter

**Способ 2 (через меню):**
- Нажмите на три точки в правом верхнем углу
- Выберите "Дополнительные инструменты" → "Расширения"

### 2. Включите режим разработчика

- В правом верхнем углу найдите переключатель **"Режим разработчика"**
- Включите его (должен стать синим)

### 3. Загрузите расширение

- Нажмите кнопку **"Загрузить распакованное расширение"**
- В открывшемся окне выберите папку `toxic-shield-extension`
- Нажмите **"Выбор папки"** (или "Select Folder")

### 4. Проверьте установку

После загрузки вы должны увидеть карточку расширения:
```
🛡️ ToxicShield Universal - Защита от токсичности
Версия: 1.0.0
Статус: Включено
```

### 5. Закрепите расширение (опционально)

- Нажмите на иконку "пазл" 🧩 в панели инструментов Chrome
- Найдите "ToxicShield Universal"
- Нажмите на "pin" 📌 чтобы закрепить иконку

## ⚙️ Шаг 4: Настройка расширения

### 1. Откройте popup расширения

- Кликните на иконку расширения 🛡️ в панели инструментов

### 2. Проверьте API URL

По умолчанию: `http://localhost:8000/api/check`

**Если вы используете удаленный сервер:**
- В поле "API URL" введите адрес вашего API
- Например: `https://your-domain.com/api/check`
- URL сохранится автоматически

### 3. Настройте чувствительность

- По умолчанию: **15%** (оптимально)
- Двигайте слайдер для изменения порога:
  - **5-15%** - блокирует только явную токсичность
  - **15-30%** - средняя чувствительность
  - **30-95%** - агрессивная фильтрация

### 4. Убедитесь, что защита включена

- Переключатель "Защита" должен быть включен (зеленый индикатор)

## ✅ Шаг 5: Проверка работы

### 1. Тестовая страница

Откройте любой сайт с комментариями, например:
- YouTube видео с комментариями
- Reddit пост
- Новостной сайт с обсуждением
- ВКонтакте пост с комментариями

### 2. Запустите сканирование

- Откройте popup расширения
- Нажмите кнопку **"🔍 Сканировать"**
- Дождитесь завершения (статус изменится)

### 3. Проверьте результаты

**В popup:**
- Счетчик "Проверено" должен увеличиться
- Если найден токсичный контент - увеличится "Заблокировано"

**На странице:**
- Токсичные комментарии будут заблюрены красным overlay
- При наведении показывается процент токсичности
- Клик по overlay покажет скрытый контент

### 4. Просмотр логов (для отладки)

**Откройте Developer Console:**
- Нажмите F12
- Перейдите во вкладку "Console"
- Ищите сообщения с префиксом `[ToxicShield]`

**Примеры логов:**
```
[ToxicShield] Initializing...
[ToxicShield] Settings loaded: {apiUrl: "...", threshold: 0.15, enabled: true}
[ToxicShield] Starting page scan...
[ToxicShield] Found 45 text elements
[ToxicShield] Toxic content found: {text: "...", score: 0.87}
[ToxicShield] Scan complete. Processed 45 elements
```

## 🐛 Устранение проблем

### Проблема: "Расширение не загружается"

**Ошибка: "Manifest file is missing or unreadable"**
- Проверьте, что файл `manifest.json` существует
- Убедитесь, что JSON синтаксис правильный
- Откройте файл в текстовом редакторе и проверьте

**Ошибка с иконками:**
```
Could not load icon 'icons/icon16.png'
```
- Создайте иконки (см. Шаг 1)
- Или временно удалите секции `"icons"` и `"default_icon"` из manifest.json

### Проблема: "API недоступен"

**Проверьте Backend:**
```bash
# Проверка статуса
curl http://localhost:8000/health

# Если не работает - проверьте Docker
docker ps
docker logs toxic-api-app
```

**Если Backend на другом URL:**
- Откройте popup расширения
- Измените API URL на правильный
- Проверьте, что URL доступен из браузера

### Проблема: "Контент не блокируется"

**1. Проверьте настройки:**
- Переключатель "Защита" включен
- API URL правильный
- Чувствительность не слишком высокая

**2. Проверьте логи:**
- Откройте Developer Console (F12)
- Ищите ошибки API запросов
- Проверьте, что сканирование запускается

**3. Ручное сканирование:**
- Откройте popup
- Нажмите "Сканировать"
- Проверьте, увеличился ли счетчик "Проверено"

**4. Проверьте Backend:**
```bash
# Тестовый запрос
curl -X POST http://localhost:8000/api/check \
  -H "Content-Type: application/json" \
  -d '{"text":"Это тестовое сообщение","threshold":0.15}'

# Ожидаемый ответ:
# {"is_toxic":false,"toxicity_score":0.01,"model_used":"chatgpt"}
```

### Проблема: "CORS ошибки"

Если видите ошибку в Console:
```
Access to fetch at 'http://localhost:8000/api/check' from origin '...' has been blocked by CORS
```

**Решение:**
- Backend уже настроен с CORS для всех доменов
- Убедитесь, что вы используете последнюю версию `toxic-api`
- Перезапустите Docker контейнер:
  ```bash
  cd toxic-api
  docker-compose restart
  ```

### Проблема: "Расширение тормозит браузер"

**Уменьшите нагрузку:**
1. Увеличьте порог чувствительности (проверяется меньше контента)
2. Временно отключите расширение на тяжелых страницах
3. Очистите кэш: откройте popup → "Сброс"

**Для больших страниц:**
- Расширение обрабатывает элементы батчами (10 элементов за раз)
- Есть задержки между батчами (100мс)
- MutationObserver имеет дебаунс (500мс)

## 🔄 Обновление расширения

При внесении изменений в код:

### 1. Обновите файлы
- Отредактируйте нужные файлы (.js, .html, .css)
- Сохраните изменения

### 2. Перезагрузите расширение
- Откройте `chrome://extensions/`
- Найдите "ToxicShield Universal"
- Нажмите кнопку **"Обновить"** (🔄)

### 3. Обновите страницу
- Перейдите на тестовую страницу
- Нажмите F5 для перезагрузки
- Расширение будет использовать новый код

## 📤 Публикация в Chrome Web Store (опционально)

Если хотите опубликовать расширение:

### 1. Подготовьте ZIP архив
```bash
cd toxic-shield-extension
zip -r ../toxic-shield-extension.zip * -x "*.git*" "README.md" "INSTALLATION.md"
```

### 2. Зарегистрируйтесь как разработчик
- Перейдите на [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
- Оплатите одноразовый взнос $5

### 3. Загрузите расширение
- Нажмите "Новый элемент"
- Загрузите ZIP файл
- Заполните описание, скриншоты, иконки
- Отправьте на модерацию (обычно 1-3 дня)
