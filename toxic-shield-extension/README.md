# 🛡️ ToxicShield Universal - Browser Extension

**Универсальное расширение для Chrome/Edge, автоматически блокирующее токсичный контент на ЛЮБЫХ сайтах**

## 🎯 Возможности

- ✅ **Работает на ВСЕХ сайтах** - YouTube, ВКонтакте, Twitter, Reddit, форумы, новостные сайты и др.
- 🌍 **Мультиязычность** - поддержка русского, казахского и английского языков
- 🤖 **AI-powered** - использует ChatGPT (gpt-4o-mini) для точного определения токсичности
- 🔄 **Динамический контент** - автоматически сканирует новые комментарии и сообщения
- 👁️ **Умное блюрирование** - токсичный контент скрывается, но можно показать по клику
- 📊 **Статистика** - отслеживание количества проверенных и заблокированных сообщений
- ⚙️ **Настройки чувствительности** - регулируемый порог токсичности (5-95%)
- 💾 **Кэширование** - быстрая работа благодаря кэшированию результатов

## 🚀 Как это работает

1. **Сканирование** - расширение автоматически находит весь текстовый контент на странице
2. **Проверка** - отправляет текст в API бэкенда для анализа через ChatGPT
3. **Блокировка** - токсичный контент блюрится с предупреждением
4. **Разблокировка** - пользователь может кликнуть, чтобы увидеть скрытый контент

## 📋 Требования

- **Браузер**: Google Chrome, Microsoft Edge или другой Chromium-браузер
- **API Backend**: Запущенный FastAPI сервер с токсичным детектором (см. [toxic-api](../toxic-api/README.md))
- **Доступ к API**: Backend должен быть доступен по URL (по умолчанию `http://localhost:8000/api/check`)

## 📦 Установка

Подробная инструкция по установке в [INSTALLATION.md](INSTALLATION.md)

**Краткая версия:**

1. Скачайте папку `toxic-shield-extension`
2. Откройте Chrome → `chrome://extensions/`
3. Включите "Режим разработчика"
4. Нажмите "Загрузить распакованное расширение"
5. Выберите папку `toxic-shield-extension`

## ⚙️ Настройка

### API URL
По умолчанию расширение подключается к `http://localhost:8000/api/check`

Чтобы изменить URL:
1. Откройте popup расширения (клик на иконку)
2. В поле "API URL" введите адрес вашего API
3. Изменения сохраняются автоматически

### Чувствительность
Регулируйте порог токсичности (15% по умолчанию):
- **Низкий (5-15%)** - блокирует только явно токсичный контент
- **Средний (15-30%)** - оптимальный баланс
- **Высокий (30-95%)** - агрессивная фильтрация

## 🎮 Использование

### Автоматический режим
После установки расширение **автоматически** работает на всех сайтах:
- Сканирует страницу при загрузке
- Отслеживает новый динамический контент
- Блокирует токсичные комментарии сразу

### Popup интерфейс
Откройте popup (клик на иконку расширения):

**Статистика:**
- Проверено - общее количество проверенных элементов
- Заблокировано - количество скрытых токсичных элементов

**Управление:**
- Переключатель "Защита" - включить/выключить расширение
- Слайдер "Чувствительность" - настройка порога токсичности
- Кнопка "Сканировать" - запустить сканирование страницы вручную
- Кнопка "Сброс" - очистить статистику и показать весь скрытый контент

### Взаимодействие с контентом
**Заблокированный контент:**
- Отображается с красным overlay
- Показывает процент токсичности
- Подсказка "Нажмите, чтобы показать"

**Клик по контенту:**
- Показывает скрытый текст с полупрозрачностью
- Добавляет красную рамку для обозначения токсичности

## 🔧 Технические детали

### Архитектура
```
toxic-shield-extension/
├── manifest.json          # Конфигурация расширения (Manifest V3)
├── content.js            # Основной скрипт сканирования (330 строк)
├── background.js         # Service Worker
├── popup.html            # Интерфейс popup окна
├── popup.js              # Логика popup
├── styles.css            # Стили для блюра и overlay
└── icons/                # Иконки расширения
```

### Permissions
- `storage` - сохранение настроек и статистики
- `activeTab` - доступ к текущей вкладке
- `<all_urls>` - работа на всех сайтах

### Content Script
**Основные функции:**
- `getTextElements()` - находит все текстовые элементы (комментарии, посты, сообщения)
- `checkToxicity(text)` - проверка через API с кэшированием
- `blurElement(element, score)` - применение блюра с overlay
- `scanPage()` - полное сканирование страницы батчами по 10 элементов
- `MutationObserver` - отслеживание динамического контента

### Кэширование
- Результаты проверок кэшируются в `Map` (до 1000 записей)
- Избегает повторных запросов к API для одинакового текста
- Обработанные элементы отмечаются через `WeakSet`

### Производительность
- Батчирование запросов (10 элементов одновременно)
- Задержка 100мс между батчами
- Дебаунс для MutationObserver (500мс)
- Ограничение длины текста (10-5000 символов)

## 🐛 Отладка

### Просмотр логов
1. Откройте Developer Tools (F12)
2. Перейдите в Console
3. Ищите сообщения с префиксом `[ToxicShield]`

### Типичные проблемы

**Расширение не работает:**
- Проверьте, что Backend API запущен и доступен
- Убедитесь, что URL API указан правильно в настройках
- Проверьте, что переключатель "Защита" включен

**API ошибки:**
- Откройте `http://localhost:8000/docs` - должна открыться документация API
- Проверьте логи Backend: `docker logs toxic-api-app`
- Убедитесь, что OpenAI API ключ настроен

**Контент не блокируется:**
- Увеличьте чувствительность (снизьте порог)
- Нажмите кнопку "Сканировать" для ручного запуска
- Обновите страницу (F5)

## 🔒 Приватность

- Расширение не собирает персональные данные
- Все тексты отправляются только на ваш собственный API сервер
- Настройки хранятся локально в браузере через `chrome.storage.sync`
- Кэш хранится только в памяти и очищается при закрытии браузера
